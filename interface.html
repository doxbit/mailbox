Interface Design.

Proposed designs for the mobile app.

Login Page

This is the first landing page when a new or returning user opens the app. It gives new users the option to sign up, or returning users the opportunity to login.

Homepage

Landing page when a user opens the app while not at a mailbox. They can compose mail, but not read or send saved mail.

Mailbox Homepage

Upon scanning a Mailbox the user is presented with options to either read mail in the inbox or send saved mail.

Send Mail Confirmation

This allows the user can review mail writen earlier, to confirm before sending it. This ensures users don't accidentally send mail unintentionally, as well as ensuring that all mail has required information (such as a delivery address).

Inbox Overview

Presents new mail to the user in a simple reverse chronological order.

Message

Allows users to read mail in a simple DOS style format. Considering adding the functionality to export mail to a basic text format.
